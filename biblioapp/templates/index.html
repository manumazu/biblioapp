Biblio(bus) est un système de localisation d'objets.

Bibliobus permet d'indexer, de partager, et de localiser des objets facilement et qui, en plus, est joli.

C’est un système d’éclairage pour rechercher et repérer, via un signal lumineux, un objet disposé sur un support. Basé sur un index interrogeable à distance et couplé à circuit de type Arduino, communiquants en liaison série (bus), il permet d'interroger un support pour trouver un objet.

L'adressage d'objets, pourquoi faire ?

La consommation et la production d'objets sont devenus cruciaux pour l'économie de distribution.

Parmi ces objets, il en est certains qui nous sont nécessaires pour vivre. Ces objets doivent demeurer échangeables, réutilisables, partageables.
Bibliobus est un moyen efficace et ludique pour inscrire le cycle de vie des objets dans une économie circulaire efficiente, pour tous.

Démos

L'application mobile "BiblioApp" permet de commander le système de localisation d'objets. Voici quelques vidéos sur le fonctionnement de l'appli et son interaction avec une bibliothèque équipée du système "Bibus" composée de 140 ouvrages (visible en incrustration).

  • Les listes de contenu

    Liste des auteurs (liste par défaut) est une navigation alphabétique des initials de noms. On peut afficher ainsi facilement tous les livres d’un auteur, en bleu (couleur par défaut).

    Liste des catégories :
    Les catégories disposent de codes couleurs qu’on peut éditer via l’application de gestion de bibliothèque sur le serveur. Les livres d’une catégorie sont indiqués dans la bibliothèque par les leds avec le code couleur concerné, sinon en bleu ...

  • La commande vocale

    La commande vocale fonctionne comme une recherche par mot clé. Elle permet d’extraire à partir d’une phrase type les mots clés à traiter. Il y a 2 types de commandes : “trouve” et “affiche”.

    Si je demande à l’appli “trouve les livres d'histoire”, le résultat sera la liste des items trouvés pour le mot clé histoire (comme si j’avais saisi “histoire” dans la recherche).

    ...

  • Les effets lumineux

    Ce sont des effets qui sont programmés dans l’arduino, mais qu’on peut commander depuis l’appli mobile. La différence entre le code personnalisé qu’on peut jouer depuis le serveur, et le code enregistré dans l’arduino est qu’ils ne sont pas de même nature.

    Le premier est exécuté de manière séquentiel par l’appli mobile et envoyé sous forme de message à l’arduino, tandis que l’autre est exécuté par l’arduino permettant des effets plus puissants (comme le fade in / fade out).
